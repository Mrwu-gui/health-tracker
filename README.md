# health-tracker
# 健康管理应用开发指南（最终版）

## 一、项目概述
- **项目名称**：健康管理工具
- **目标平台**：PC端 + 微信小程序
- **技术栈**：
  - 前端：Vue 3 + Element Plus（PC端）、uniapp（小程序端）
  - 后端：Spring Boot + MyBatis-Plus + MySQL
  - 数据库：MySQL（存储用户信息、记录、目标、药物信息）
  - 缓存/消息：Redis（可选，用于提醒消息缓存）
  - 图表：ECharts（数据可视化）
- **功能定位**：帮助用户记录和管理健康数据（运动、饮食、睡眠、体重、血压、药物等），提供统计分析、目标管理和提醒功能，面向普通用户及中老年群体。

---

## 二、目标用户
- 普通健康管理需求用户（记录运动、饮食、睡眠）
- 中老年用户（关注药物管理和健康提醒）
- 需要统计分析、目标管理和跨端同步的用户

---

## 三、核心功能模块

### 1. 用户管理
- 注册 / 登录（支持微信授权登录）
- 个人信息管理（性别、年龄、身高、体重、健康状况等）
- 账户安全：密码加密存储，JWT 登录验证
- 可选：家庭成员管理（共享健康数据）

### 2. 数据记录
- **运动记录**：步数、跑步距离、运动时长、消耗卡路里
- **饮食记录**：餐次、食物、热量、营养信息
- **睡眠记录**：睡眠时长、深浅睡比例
- **体重管理**：体重变化记录，BMI 计算
- **血压/心率**（可选）
- **药物记录**：药物名称、剂量、服用频率、开始结束时间（中老年特色功能）

### 3. 目标管理
- 用户可设定每日/每周运动目标、饮食目标、睡眠目标
- 提供完成度统计（百分比/进度条）
- 支持提醒功能（微信小程序推送或 PC 弹窗）

### 4. 健康分析
- 数据可视化展示：
  - 体重变化图
  - 步数/运动时间统计
  - 睡眠质量图表
  - 药物服用完成率
- 统计周期：日/周/月
- 提供分析结论和健康建议（智能分析，可后期加入 AI 功能）

### 5. 提醒与通知
- 日常运动提醒、饮食提醒、睡眠提醒
- 药物服用提醒（中老年用户特色功能）
- 微信小程序订阅消息接口
- PC端可通过弹窗或桌面通知
- 支持提前提醒（如 10 分钟前）

### 6. 报告生成
- 自动生成周报/月报
- 支持导出 PDF 或分享小程序
- 展示用户健康趋势和目标完成情况

### 7. 药物管理（中老年特色功能）
- 添加药物信息（药名、剂量、用法、备注）
- 设置每日/按需服药提醒
- 记录服药情况（已服/未服）
- 统计服药完成率和趋势
- 异常提醒（漏服、重复服药）

### 8. 社区 / 分享（可选）
- 用户分享健康记录、运动成果、健康心得
- 后期可扩展互动功能，如健康问答、打卡排行榜

### 9. AI / 智能健康分析（可扩展）
- 智能生成健康建议（运动量、饮食调整、睡眠优化）
- 可接入 OpenAI API 或自研算法
- 可根据用户数据推荐个性化健康计划

## 四、技术架构

### 前端
- **PC端**：Vue 3 + Element Plus
  - 登录/注册页面
  - 健康数据管理面板
  - 数据统计图表页面
  - 目标管理页面
  - 用户信息设置页面
- **小程序端**：uniapp
  - 首页：快速记录（运动、饮食、睡眠、药物）
  - 数据统计页：查看图表和健康报告
  - 目标页：查看目标完成情况
  - 提醒页：消息和服药提醒
  - 服药打卡按钮：快速记录“已服”

### 后端
- Spring Boot + MyBatis-Plus
- RESTful API
- 数据库：MySQL
- 可选缓存：Redis
- 消息推送：微信订阅消息 / PC端弹窗

## 五、开发注意事项
1. M2 芯片 Mac 开发注意：
   - 使用 Apple Silicon 原生版本（VS Code、Node、Java）
2. 数据库操作：
   - 使用 MyBatis-Plus 提供的通用 CRUD 和 Lambda 查询
3. 消息推送：
   - 微信小程序端需申请订阅消息权限
4. 数据可视化：
   - 使用 ECharts 实现图表
5. 开发流程：
   - 先实现核心模块（数据记录、目标、提醒）
   - 再扩展智能分析、药物管理、报告生成、社区功能
